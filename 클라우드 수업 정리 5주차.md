## 1. 보안 그룹 생성 (Security Group, SG)

### 경로

`EC2 → 네트워크 및 보안 → 보안 그룹 → 보안 그룹 생성`

### 설정 항목

| 항목 | 값 예시 |
| --- | --- |
| 보안 그룹 이름 | `sgu-202601-EC2` (생성 후 이름 변경 불가) |
| 설명 | `sgu-202601 EC2 Security Group` |
| VPC | 사용할 VPC 선택 |

### 인바운드 규칙

- 유형: **SSH** / 프로토콜: TCP / 포트: 22 / 소스: 0.0.0.0/0

### 아웃바운드 규칙

- 유형: **모든 트래픽** / 프로토콜: 전체 / 포트: 전체 / 대상: 0.0.0.0/0

## 3. 키페어 생성

### 키페어 파일 형식

| OS | 형식 |
| --- | --- |
| Windows | `.ppk` (PuTTY Private Key) |
| Mac / Linux | `.pem` (Privacy-Enhanced Mail) |

> ⚠️ **주의:** 다운로드 받은 ppk/pem 파일은 반드시 **개인 메일로 전송**하여 보관할 것!
> 

### PEM과 PPK 개념

- **PEM (.pem):** Linux, macOS에서 기본 지원. `ssh` 명령어로 사용.
- **PPK (.ppk):** Windows의 SSH 클라이언트 **PuTTY**에서 사용.

---

## 4. PEM 키 → PPK 변환 방법 (Windows)

1. [https://www.putty.org/](https://www.putty.org/) 접속
2. `Download Putty` → `putty.zip` 다운로드
3. 압축 해제 후 **PUTTYGEN.EXE** 실행
4. `Load` 클릭 → 파일 형식을 `All Files (*.*)` 로 변경 → `.pem` 파일 선택 → 열기
5. 확인 팝업에서 **확인** 클릭
6. `Save private key` 클릭 → 경고창에서 **예(Y)** 클릭 → `.ppk` 파일로 저장

## 5. EC2 인스턴스 개념

- **EC2:** 가상 서버를 제공하는 AWS 서비스
- **인스턴스:** EC2에서 생성한 개별 가상 서버

## 6. 인스턴스 상태 (수명 주기)

| 상태 | 설명 |
| --- | --- |
| `pending` | 대기 중 (최초 시작 직후) |
| `running` | 실행 중 |
| `stopping` | 중지 중 |
| `stopped` | 중지됨 |
| `rebooting` | 재부팅 |
| `shutting-down` | 종료 중 |
| `terminated` | 종료됨 (삭제) |

### 수명 주기 흐름

```
최초 시작 → 대기 중 → 실행 중 → 중지 중 → 중지됨
                              ↓               ↓
                           종료 중 ←←←←←←  종료
                              ↓
                           종료됨
```

> 재부팅: 실행 중 ↔︎ 재부팅 (양방향)
>

## 7. EC2 접속 방법 (Windows / PuTTY)

1. 인스턴스의 **퍼블릭 IPv4 주소** 복사 (인스턴스 요약 탭에서 확인)
2. **PUTTY.EXE** 실행
3. `Session` → **HostName**에 IP 주소 입력
4. `Connection → SSH → Auth → Credentials → Browse` → `.ppk` 파일 선택
5. `Open` 클릭
6. 보안 경고 팝업 → **Accept** 클릭
7. `login as:` 에 **`ec2-user`** 입력하여 접속

## 8. Security Group 수정 (인바운드 규칙 편집)

### 경로

`인스턴스 선택 → 보안 탭 → 보안 그룹 클릭 → 인바운드 규칙 편집`

### ICMP 규칙 추가 (ping 허용)

- `규칙 추가` 클릭
- 유형: **모든 ICMP - IPv4**
- 소스: **내 IP**
- `규칙 저장` 클릭

> ICMP 규칙 추가 후 `ping [EC2 IP] -t` 명령으로 응답 확인 가능
>

## 9. 인스턴스 유형 변경

- 유형 변경은 **인스턴스가 중지된 상태**에서만 가능

- 예) `t3.nano` → `t3.micro`

## 10. EC2 모니터링 (CloudWatch)

### CloudWatch란?

AWS 리소스를 실시간으로 관찰·기록·대응하는 모니터링 서비스

| 핵심 기능 | 설명 |
| --- | --- |
| **Metrics (지표)** | 숫자로 보는 상태 데이터 |
| **Logs (로그)** | 텍스트 형태의 기록 |
| **Alarm (알람)** | SNS 연동으로 자동 알림 |
| **Dashboard (대시보드)** | 모든 상태를 한 화면에서 확인 |

## 11. EC2 세부 모니터링 활성화

### 경로

`인스턴스 선택 → 모니터링 탭 → 세부 모니터링 관리 → 활성화 체크 → 확인`

> 세부 모니터링 활성화 시 **1분 단위** 모니터링 그래프 확인 가능 (추가 요금 발생)
>

## 12. CPU 사용률 CloudWatch 대시보드 추가

1. 인스턴스 → 모니터링 탭 → **CPU 사용률** 확장
2. 우측 상단 아이콘 클릭 → **지표에서 보기** 클릭
3. CloudWatch Metrics 화면에서 `작업 → 대시보드에 추가` 클릭
4. **새로 생성** → 대시보드 이름 입력 (예: `sgu-202601-cloudwatch`) → **생성**
5. **대시보드에 추가** 클릭 → **대시보드에 저장** 클릭
