## 1. S3 정적 웹 호스팅 실습

### 실습 목표

- S3 버킷을 활용해 정적 웹사이트를 호스팅한다.
- `index.html` 및 이미지를 S3에 업로드하고, 퍼블릭 접근을 설정한다.

---

### 실습 순서

### ① 퍼블릭 액세스 허용

- 버킷 → **권한** 탭 → **편집** 버튼 클릭
- "모든 퍼블릭 액세스 차단" 체크 해제 후 **변경 사항 저장**

---

### ② index.html 생성

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>S3 정적 웹사이트 실습</title>
</head>
<body>
  <h1>Amazon S3 정적 웹사이트 호스팅 실습</h1>
  <p>이 페이지는 S3 버킷에서 호스팅되고 있습니다.</p>
  <h2>업로드한 이미지:</h2>
  <img src="class/img/image.png" alt="S3에 업로드한 이미지">
</body>
</html>
```

---

### ③ 파일 업로드 구조

| 파일 | 업로드 위치 |
| --- | --- |
| `index.html` | 버킷 루트 (`/`) |
| `image.png` | `class/img/` 폴더 |
- S3 콘솔에서 **폴더 만들기** 버튼으로 `class` → `img` 폴더 생성 후 업로드

---

### ④ index.html 접근 시도 → 오류 발생

- 객체 URL 직접 접근 시 `AccessDenied` 오류 발생
- **원인**: 버킷 정책(Bucket Policy)이 없기 때문

---

### S3 버킷 정책(Bucket Policy)

### 개념

- S3 리소스(버킷 및 객체)에 대한 **접근 제어 정책**
- **JSON 형식** 문서
- 버킷 단위로 적용 → 버킷 내 모든 객체에 적용 가능
- IAM 정책과 달리 **리소스 자체에 붙이는 정책** (Resource-based Policy)

### IAM 정책 비교

| 구분 | 설명 |
| --- | --- |
| **Identity-based Policy** | IAM 사용자/그룹/역할에 부여. "누가 무엇을 할 수 있는가" |
| **Resource-based Policy** | S3, SQS 등 리소스에 직접 부여. "이 리소스에 누가 접근 가능한가" |

> 기본적으로 모든 객체는 **Private(비공개)** 상태 → 퍼블릭 접근은 **명시적으로 허용** 필요
> 

---

### 버킷 정책 예시

### 전체 버킷 공개 (기본)

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicWebsiteAllow",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::버킷이름/*"
    }
  ]
}
```

### 특정 파일/폴더만 허용

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicWebsiteAllow",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": [
        "arn:aws:s3:::버킷이름/index.html",
        "arn:aws:s3:::버킷이름/class/img/*"
      ]
    }
  ]
}
```

### 특정 IP만 접근 허용

```json
{
  "Condition": {
    "IpAddress": {
      "aws:SourceIp": "123.123.123.123"
    }
  }
}
```

### 특정 폴더 다운로드 차단

```json
{
  "Sid": "DenyGetObject",
  "Effect": "Deny",
  "Principal": "*",
  "Action": "s3:GetObject",
  "Resource": "arn:aws:s3:::버킷이름/class/test/*"
}
```

> `"Effect": "Deny"` + `"Action": "s3:GetObject"` 조합으로 다운로드만 차단 가능
>

## 2. AWS CLI 기반 EC2 + S3 자동화 실습

### 실습 목표

- AWS CLI 명령어 사용법 습득
- EC2 **UserData 스크립트**를 통한 부팅 시 자동 구성
- S3에서 파일을 자동으로 EC2로 다운로드

---

### 실습 구성도

```
EC2 시작 → UserData 실행 → (AWS CLI로 S3에서 index.php 다운로드) → EC2 시작 완료
```

---

### S3에 index.php 업로드

```php
<html>
<head><title>AWS EC2 PHP</title></head>
<body>
  <h1>Welcome to the AWS EC2 PHP</h1>
  <?php
    print "The Current Date and Time is: <br/>";
    print date("g:i A l, F j, Y.");
  ?>
  <?php phpinfo(); ?>
</body>
</html>
```

---

### EC2 생성 설정

| 항목 | 값 |
| --- | --- |
| 이름 | `sgu-계정-ec2-s3test` |
| OS 이미지 | Amazon Linux 2 |
| 인스턴스 타입 | t3.nano |
| 키페어 / 보안그룹 | 자신이 생성한 것 사용 |
| IAM 인스턴스 프로파일 | 자신의 계정에 해당하는 프로파일 선택 |

---

### UserData 스크립트

> EC2 생성 → **고급 세부 정보** → **사용자 데이터** 입력
> 

```bash
#!/bin/bash
yum install httpd php -y
aws s3 cp s3://버킷이름/index.php /var/www/html --region ap-northeast-2
systemctl restart httpd
```

- `httpd`, `php` 설치
- S3에서 `index.php`를 EC2의 웹 루트(`/var/www/html`)로 복사
- Apache 웹서버 재시작

---

### AWS CLI란?

> **Amazon Web Services Command Line Interface**
> 
> - 터미널/명령 프롬프트로 AWS 리소스(S3, EC2, IAM 등)를 제어하는 도구
> - 콘솔 클릭 없이 명령어만으로 S3 버킷 생성, 파일 업로드, EC2 실행 등 가능

---

### AWS S3 CLI 주요 명령어

| 명령어 | 설명 |
| --- | --- |
| `aws s3 ls` | 버킷 목록 확인 |
| `aws s3 ls s3://버킷이름/` | 버킷 내 파일 목록 확인 |
| `aws s3 cp 로컬경로 s3://버킷이름/경로` | 파일 **업로드** |
| `aws s3 cp s3://버킷이름/경로 로컬경로` | 파일 **다운로드** |
| `aws s3 cp s3://버킷/폴더 ./로컬폴더 --recursive` | 폴더 전체 **다운로드** |
| `aws s3 cp ./로컬폴더 s3://버킷/폴더 --recursive` | 폴더 전체 **업로드** |

### 예시

```bash
# 파일 업로드
aws s3 cp test.log s3://my-bucket

# 파일 다운로드
aws s3 cp s3://my-bucket/test.txt .

# 폴더 전체 다운로드
aws s3 cp s3://my-bucket/folder ./local-folder --recursive

# 폴더 전체 업로드
aws s3 cp ./local-folder s3://my-bucket/folder --recursive
```

## 핵심 정리

| 개념 | 요약 |
| --- | --- |
| S3 정적 호스팅 | 버킷에 html/이미지 업로드 + 퍼블릭 액세스 허용 + 버킷 정책 추가 |
| 버킷 정책 | JSON 형식, 리소스 기반 접근 제어, Effect/Action/Resource/Condition 구성 |
| UserData | EC2 최초 부팅 시 자동 실행되는 스크립트 (설치, S3 다운로드, 서비스 시작 등) |
| AWS CLI | 터미널로 AWS 리소스를 제어하는 도구, `aws s3 cp/ls` 등 활용 |
| IAM 인스턴스 프로파일 | EC2가 S3 등 AWS 리소스에 접근할 수 있도록 권한을 부여하는 역할 |
