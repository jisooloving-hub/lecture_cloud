# 클라우드 수업 정리 4주차

# Git 중급 - Branch 노트

---

## 1. Branch란?

- 특정 시점에서 코드의 버전(상태)을 나누어 별도로 관리할 수 있도록 하는 기능
- 기본 브랜치(Main, Master)에서 나뭇가지처럼 새로운 브랜치를 생성하여 기능 추가, 오류 수정 등을 가능하게 함
- 분기된 가지: 특정 시점에서 갈라져서 독립적인 작업 가능

---

## 2. Branch 명령어

| 기능 | 명령어 |
| --- | --- |
| 브랜치 목록 확인 | `git branch` |
| 브랜치 생성 | `git branch <브랜치명>` |
| 브랜치 이동 | `git switch <브랜치명>` |
| 브랜치 생성 + 이동 | `git switch -c <브랜치명>` |
| 브랜치 이름 수정 | `git branch -m <기존 브랜치명> <새 브랜치명>` |
| 브랜치 삭제 | `git branch -d <브랜치명>` |

---

## 3. Branch 병합하기 - Merge

leader 브랜치를 main 브랜치로 merge하는 방법:

1. main 브랜치로 이동
2. `git merge leader` 실행
3. `git log`로 커밋 내역 확인

---

## 4. Merge 취소하기

- 병합 이전 커밋 해시로 reset:
    
    ```
    git reset --hard <커밋 해시>
    ```
    

---

## 5. 병합된 브랜치 삭제

```
git branch -d leader
```

---

## 6. Branch 병합하기 - Rebase

- **rebase**: 기반을 다시 설정하다
- manager 브랜치를 main 브랜치로 rebase하면, manager 브랜치의 커밋들이 main의 최신 커밋 위에 다시 쌓임

### Rebase 절차

1. manager 브랜치로 이동
2. `git rebase main` 실행
3. main 브랜치로 이동
4. `git merge manager` 실행 (Fast-forward)
5. manager 브랜치 삭제

> **주의!** rebase 이후 main 브랜치가 뒤처져 있으므로, 반드시 main에서 `git merge <브랜치명>` 을 추가로 실행해야 함
> 

### Rebase 중 충돌 발생 시

```
git rebase --abort   # rebase 전 상태로 복구
```

## 7. Branch 병합 중 충돌! - Merge

### 충돌 없는 merge 절차 (p.23)

1. main 브랜치에 `conflict-1.txt` 파일 생성 후 add & commit
    
    ```
    dept: ITsoftware
    member:
    - Lee
    ```
    
2. member에 Park 추가 후 add & commit
3. dev 브랜치 생성
4. dev 브랜치에서 Woo 추가 후 add & commit
5. dev 브랜치에서 Kim 추가 후 add & commit
6. main 브랜치로 이동 후 파일 확인
7. dev 브랜치 병합: `git merge dev`
8. 확인 후 dev 브랜치 삭제

### 충돌이 발생하는 merge 절차 (p.24)

1. test 브랜치 생성 → test 브랜치 이동
2. test 브랜치: member에 Min 추가 후 add & commit
3. main 브랜치: member에 Son 추가 후 add & commit
4. main 브랜치에서 test 브랜치와 merge 시도: `git merge test`

충돌 발생 예시:

```
<<<<<<< HEAD (Current Change)
- Son
=======
- Min
>>>>>>> test (Incoming Change)
```

### 충돌 해결 방법 (p.25)

| 방법 | 명령어 |
| --- | --- |
| merge 취소 | `git merge --abort` |
| merge 해결 | 원하는 방향으로 파일 수정 후, `git add` → `git commit` |

---

## 8. Branch 병합 중 충돌! - Rebase

### 충돌이 발생하는 rebase 절차 (p.26)

1. test2 브랜치 생성
2. test2 브랜치: member에 Choi 추가 후 add & commit
3. main 브랜치로 이동 후, member에 Ha 추가 후 add & commit
4. test2 브랜치로 이동: `git rebase main` → 충돌 발생!

충돌 발생 예시:

```
<<<<<<< HEAD (Current Change)
- Ha
=======
- Choi
>>>>>>> 커밋해시 (Choi add) (Incoming Change)
```

### 충돌 해결 방법 (p.27)

| 방법 | 명령어 |
| --- | --- |
| rebase 취소 | `git rebase --abort` |
| 충돌 해결 후 rebase 계속 진행 | 파일 수정 후 `git add` → `git rebase --continue` |
- 충돌 해결 후 add 예시 (conflict-1.txt를 원하는 내용으로 수정):
    
    ```
    dept: ITsoftware
    member:
    - Lee
    - Park
    - Woo
    - kim
    - Son
    - Min
    - Ha
    - Choi
    ```
    
- 이후 `git rebase --continue` 실행 → rebase 성공
